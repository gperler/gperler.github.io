Siesta PHP
Stored procedure based object relational mapper for php

Overview

Siesta is an object relational mapper based on stored procedures for PHP.
There are many good orm for php available. So why write a new one?

  • First of all : it is fun
  • Stored procedures are fast, that is why every interaction with the database is done via Stored procedures

Installation

Installation with composer

composer require gm314/siesta

Create configuration file with

vendor/bin/siesta init

Entities

Defining an entity

An entity always represents on table in the database. This will result in a generated class that provides functionality for loading and storing the entity in the database table defined Sample of Entity:

Name Required default description
name yes Name of the class that is generated
namespace no Namespace of the class to generate
constructClass no name of the class Name of the class to instantiate
constructNamespace no namespace of the class to generate Namespace of the class to instantiate
table no name of the class Name of the database table
delimit no false Allows to track history of the table
targetPath no . target path where to put generated classes 
<?xml version="1.0" encoding="UTF-8"?>
<entityList>
    <entity name="Artist" namespace="siestaphp\tests\functional\delimit\gen" targetPath="gen/">
        <attribute name="id" type="int" dbType="INT" primaryKey="true" required="true" autoValue="autoincrement"/>
        <attribute name="bool" type="bool" dbType="SMALLINT"/>
        <attribute name="int" type="int" dbType="INT"/>
        <attribute name="float" type="float" dbType="FLOAT"/>
        <attribute name="string" type="string" dbType="VARCHAR(100)"/>
    </entity>
</entityList>

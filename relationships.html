References

References allow to define foreign key constraints. They must have a name. The foreign class defines which Entity is referenced. Siesta will automatically add a column for every primary key that the referenced table has. 

    <entity name="ArtistEntity" namespace="siestaphp\tests\functional\reference\gen" targetPath="gen">
        <attribute name="id" type="int" dbType="INT" primaryKey="true" autoValue="autoincrement"/>
        <attribute name="name" type="string" dbType="VARCHAR(100)"/>

        <reference name="label" foreignClass="LabelEntity" required="false"/>
    </entity>

    <entity name="LabelEntity" namespace="siestaphp\tests\functional\reference\gen" targetPath="gen">
        <attribute name="id" type="int" dbType="INT" primaryKey="true" autoValue="autoincrement"/>
        <attribute name="name" type="string" dbType="VARCHAR(100)"/>
        <attribute name="city" type="string" dbType="VARCHAR(100)"/>
    </entity>

The relationship will generate these methods into the ArtistEntity. getLabel allows to retrieve the referenced object. A reload from the database can be enforced by the $forceReload parameter.

For every Primary Key column in the referenced table a getter and setter for the corresponding foreign key column will be created. In this case the LabelEntity class only uses one primary key column. 

    /**
     * @param bool $forceReload
     *
     * @return LabelEntity
     */
    public function getLabel($forceReload = false) { ... }

    /**
     * @return int
     */
    public function getLabelId() { ... }


    /**
     * @param LabelEntity $object
     */
    public function setLabel($object) { ... }

    /**
     * @param int $id
     */
    public function setLabelId($id) { ... }

Bidirectional references

You can also define bidirectional 1:1 references by adding a relationName attribute to the reference element. 

    <entity name="Customer" namespace="siestaphp\tests\functional\bidirectional\gen" table="CUSTOMER" targetPath="gen">
        <attribute name="id" type="int" dbName="ID" dbType="INT" primaryKey="true" autoValue="autoincrement"/>
        <attribute name="name" type="string" dbName="D_STRING" dbType="VARCHAR(100)"/>
        <attribute name="dob" type="DateTime" dbName="D_DATETIME" dbType="DATETIME"/>

        <reference name="address" foreignClass="Address" relationName="_customer_address_"/>
    </entity>

    <entity name="Address" namespace="siestaphp\tests\functional\bidirectional\gen" table="Address" targetPath="gen">
        <attribute name="id" type="int" dbName="ID" dbType="INT" primaryKey="true" autoValue="autoincrement"/>
        <attribute name="street" type="string" dbName="STREET" dbType="VARCHAR(100)"/>
        <attribute name="city" type="string" dbName="CITY" dbType="VARCHAR(100)"/>
        <reference name="customer" foreignClass="Customer" relationName="_customer_address_"/>
    </entity>

Siesta will set the foreign keys accordingly. You only need to disalbe the foreign key checks from the database before persisting 

   $customer = new Customer();
    $customer->setName("Columbo");

    $address = new Address();
    $address->setCity("Berlin");

    $customer->setAddress($address);

    // disable the foreign key checks before storing
    $connection = ConnectionFactory::getInstance()->getConnection();
    $connection->disableForeignKeyChecks();
    $customer->save(true);
    $connection->enableForeignKeyChecks();

1-N Collector

A One to Many Collector will collect all entities that reference the own entity. The following example shows that the ArtistEntity references 0 or 1 LabelEntity objects. The label entity defines a collector with the name artistList that collects ArtistEntity objects. It defines its foreign class and the name of the reference. 

    <entity name="ArtistEntity" namespace="siestaphp\tests\functional\collector1n\gen" table="ARTIST" targetPath="gen/">
        <attribute name="id" type="int" dbName="ID" dbType="INT" primaryKey="true" autoValue="autoincrement"/>
        <attribute name="name" type="string" dbName="D_STRING" dbType="VARCHAR(100)"/>
        <attribute name="dob" type="DateTime" dbName="D_DATETIME" dbType="DATETIME"/>

        <reference name="label" foreignClass="LabelEntity" required="false"/>
    </entity>

    <entity name="LabelEntity" namespace="siestaphp\tests\functional\collector1n\gen" table="LABEL" targetPath="gen">
        <attribute name="id" type="int" dbName="ID" dbType="INT" primaryKey="true" autoValue="autoincrement"/>
        <attribute name="name" type="string" dbName="NAME" dbType="VARCHAR(100)"/>
        <attribute name="city" type="string" dbName="CITY" dbType="VARCHAR(100)"/>

        <collector name="artistList" type="1n" foreignClass="ArtistEntity" referenceName="label"/>
    </entity>

The following methods will be generated in the LabelEntity class. They allow to collect all Artist, that reference the LabelObject. Furthermore all ArtistEntiy Object referencing this label can be deleted or a ArtistEntity object can be added to the list. 

    /**
     * @param bool $forceReload
     * @param string $connectionName
     *
     * @return ArtistEntity[]
     */
    public function getArtistList($forceReload = false, $connectionName = null) { ... }

    /**
     * @return void
     */
    public function deleteAllArtistList() { ... }

    /**
     * @param ArtistEntity $object
     *
     * @return void
     */
    public function addToArtistList(ArtistEntity $object) { ... }

Using Filter for collectors

A collector can define filters. They allow to only collect those entities that match the filter. The following filter will check that the name of the referenced object matches the a certain name. The filter defines parameter that will be available when executing the query. 

    <collector name="artistList" type="1n" foreignClass="ArtistEntity" referenceName="label">
        <filter name="ByName" filter="name LIKE P_NAME">
            <parameter name="name" spName="P_NAME" type="string" dbType="VARCHAR(100)"/>
        </filter>
    </collector>

The following method will be generated in the LabelEntity class. It accepts the defined parameter in the method and will executed the generated stored procedure. 

    /**
     * @param string $name
     * @param string $connectionName
     *
     * @return ArtistEntity[]
     */
    public function getArtistListFilterByName($name, $connectionName = null) { ... }
